# rinka example

A minimal Hono app on Cloudflare Workers using `rinka` for opt-in per-route Dynamic Worker delegation.

## Structure

- `src/index.tsx` — host Hono app
- `src/routes/index.tsx` — dynamic route (`/`) that fetches posts from JSONPlaceholder and renders them with JSX
- `src/routes/post.tsx` — inline route (`/post/:id`) handled by the host worker
- `src/generated/dynamic-manifest.ts` — generated by `rinkaVitePlugin`

## Scripts

```bash
pnpm dev        # start the Vite dev server
pnpm build      # typecheck and build
pnpm check      # typecheck only
pnpm deploy     # deploy with wrangler
```

## What it demonstrates

The `/` route is wrapped with `dynamic()`, so the JSONPlaceholder fetch and JSX rendering run inside a separate Worker isolate loaded via the `LOADER` Worker Loader binding. The `/post/:id` route is a normal inline Hono route.
