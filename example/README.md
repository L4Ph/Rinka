# rinka example

A minimal Hono app on Cloudflare Workers using `rinka` for opt-in per-route Dynamic Worker delegation.

## Structure

- `src/index.tsx` — host Hono app
- `src/routes/index.tsx` — dynamic route (`/`) that lists ramen shops from Ramen API
- `src/routes/shops.tsx` — dynamic route (`/shops/:id`) showing shop details and photos
- `src/routes/photos.tsx` — dynamic route (`/shops/:id/photos/:index`) showing a single photo
- `src/routes/prefectures.tsx` — dynamic route (`/prefectures/:prefecture/shops`) listing shops by prefecture
- `src/routes/about.tsx` — inline route (`/about`) handled by the host worker
- `src/generated/dynamic-manifest.ts` — generated by `rinkaVitePlugin`

## Scripts

```bash
pnpm dev        # start the Vite dev server with Cloudflare Workers runtime
pnpm build      # typecheck and build
pnpm check      # typecheck only
pnpm deploy     # build and deploy with wrangler
```

## What it demonstrates

Most routes are wrapped with `dynamic()`, so external API fetches and JSX rendering run inside separate Worker isolates loaded via the `LOADER` Worker Loader binding. The `/about` route is a normal inline Hono route.
